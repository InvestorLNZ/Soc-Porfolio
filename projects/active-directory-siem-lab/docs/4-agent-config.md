# Phase 4 — Wazuh Agent Configuration

## Overview

Windows Server 2022 Domain Controller connected to Wazuh as a monitored endpoint. The agent was installed on the DC and registered with the Wazuh manager using automatic key registration — enabling security event forwarding from Active Directory to the SIEM.

Both machines are on the same home network (`172.20.10.x` subnet). The Windows Server VM (VirtualBox on laptop) communicates directly with the Ubuntu desktop running Wazuh over the local network.

Connectivity verified before installation:


## Agent Installation

### Step 1 — Download Agent MSI

On the Windows Server VM, run in PowerShell as Administrator:

```powershell
Invoke-WebRequest -Uri https://packages.wazuh.com/4.x/windows/wazuh-agent-4.14.4-1.msi -OutFile $env:tmp\wazuh-agent.msi
```

### Step 2 — Install Agent

```powershell
msiexec.exe /i $env:tmp\wazuh-agent.msi /q WAZUH_MANAGER='172.20.10.10' WAZUH_AGENT_NAME='soc-lab-dc'
```

### Step 3 — Automatic Key Registration

Instead of manually copying an authentication key between machines, the `agent-auth` tool was used for automatic registration:

```powershell
cd "C:\Program Files (x86)\ossec-agent"
.\agent-auth.exe -m 172.20.10.10
```

Output confirmed:
```
INFO: Requesting a key from server: 172.20.10.10
INFO: Valid key received
```

### Step 4 — Configure Manager IP via GUI

The Wazuh Agent Manager GUI was used to set the Manager IP to `172.20.10.10` and restart the service.

### Step 5 — Start Agent

```powershell
NET START Wazuh
```

## Issues Encountered

**Internet access on Windows Server VM** — The VM was set to VirtualBox Internal Network adapter, blocking internet access needed to download the agent. Resolved by changing the adapter to NAT in VirtualBox settings.

**Duplicate agent registration error** — Running `agent-auth` a second time returned `ERROR: Duplicate agent name`. This is expected — the agent was already registered from the first successful run. Resolution: simply start the Wazuh service without re-running agent-auth.

**Connection reset by manager** — After registration, the dashboard showed "Connection reset by manager". Resolved by restarting all Wazuh services on the Ubuntu machine using `sudo /var/ossec/bin/wazuh-control restart`, then restarting the Windows agent service.
